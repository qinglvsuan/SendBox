## 🌟 什么是 SendBox？

**SendBox** 是一款专为局域网内设备间极速传输文件与文本而设计的轻量级应用。
无论你在哪个设备上需要传输文件或文本，只需将它们暂存到 SendBox，其他设备即可随时取用，随用随走。

---

## ✨ 核心特点

| 特性 | 说明 |
|---|---|
| 📂 **文件暂存** | 区别于常见p2p机制，采用暂存盘机制，可以操作完一端再操作另一端，不必同时操作双端 |
| 🔗 **WEB支持** | 支持web访问，仅需要单端下载软件，即可双端互传 |
| 🌐 **跨平台** | Android + Windows 双端，同一局域网无缝互传 |

---

## 🛠️ 技术架构

- **UI 层**: Flutter (Dart) 构建跨平台极简高颜值 UI。
- **核心逻辑层**: Rust (基于 `axum` HTTP 服务和 `tokio` 异步并发) 通过 `flutter_rust_bridge` (v2) 接入。
- **文件管理**: 内存暂存与流式分发，支持解除上传大小限制的授权弹窗询问机制。
- **平台特性**: 使用 `desktop_drop` 支持 Windows 端文件原生拖拽，`device_info_plus` 自动获取系统名称。

## 📖 使用指南 (Usage)

### 场景 A: App-to-App (双端均安装 SendBox 客户端)
1. **启动应用**: 在发送端和接收端分别打开 SendBox。局域网内的设备将会自动发现彼此。
2. **选择文件/文本**: 在发送端点击“选择文件”或直接拖拽文件进入窗口，也可以输入需要同步的文本。
3. **接收提取**: 在接收端的发现列表中，点击发送端设备的名称，即可无缝提取刚刚暂存的内容。

### 场景 B: App-to-Web (接收端仅使用浏览器)
1. **启动应用**: 在发送端打开 SendBox 客户端。
2. **获取 Web 地址**: 在发送端界面上会显示本机的服务网址（例如 `http://192.168.1.100:8080`）。
3. **浏览器访问**: 在接收端的任意现代浏览器中输入该地址，即可进入 Web 交互界面。
4. **互传交互**: 
    - **下载**: 在 Web 端可以直接看到 App 端暂存的文件和文本，点击即可下载。
    - **上传**: 在 Web 端点击上传文件，App 端会弹出传输请求授权，点击“允许”即可完成无限制大小的文件双向传输。

## 📦 如何编译构建

1. 确保已正确配置 [Flutter](https://flutter.dev/) 和 [Rust](https://rustup.rs/) 开发环境。
2. 克隆本仓库后进入项目目录。
3. 生成跨语言桥接代码 (FRB)：
   ```bash
   flutter_rust_bridge_codegen generate
   ```
4. 构建各平台应用：
   ```bash
   flutter build windows
   flutter build apk --split-per-abi
   ```
